**OpenClaw** 是一个可以运行在你自己设备上的_个人 AI 助手_。
它可以在你已经使用的渠道中回复你（WhatsApp、Telegram、Slack、Discord、Google Chat、Signal、iMessage、BlueBubbles、IRC、Microsoft Teams、Matrix、Feishu、LINE、Mattermost、Nextcloud Talk、Nostr、Synology Chat、Tlon、Twitch、Zalo、Zalo Personal、WebChat）。它支持 macOS/iOS/Android 语音输入输出，也可以渲染你可控制的实时 Canvas。Gateway 只是控制平面，核心产品是助手本身。

如果你希望一个单用户、始终在线、响应快、像本地应用一样的个人助手，这就是它。

[官网](https://openclaw.ai) · [文档](https://docs.openclaw.ai) · [愿景](VISION.md) · [DeepWiki](https://deepwiki.com/openclaw/openclaw) · [快速开始](https://docs.openclaw.ai/start/getting-started) · [升级](https://docs.openclaw.ai/install/updating) · [案例展示](https://docs.openclaw.ai/start/showcase) · [FAQ](https://docs.openclaw.ai/help/faq) · [向导](https://docs.openclaw.ai/start/wizard) · [Nix](https://github.com/openclaw/nix-openclaw) · [Docker](https://docs.openclaw.ai/install/docker) · [Discord](https://discord.gg/clawd)

## 安装（推荐）

运行时要求：**Node ≥22**。

```bash
npm install -g openclaw@latest
# 或：pnpm add -g openclaw@latest

openclaw onboard --install-daemon
```

向导会安装 Gateway 后台服务（launchd/systemd user service），确保其持续运行。

## 快速开始（TL;DR）

运行时要求：**Node ≥22**。

完整新手指南（认证、配对、渠道）：[Getting started](https://docs.openclaw.ai/start/getting-started)

```bash
openclaw onboard --install-daemon

openclaw gateway --port 18789 --verbose

# 发送一条消息
openclaw message send --to +1234567890 --message "Hello from OpenClaw"

# 与助手对话（可回送到任意已连接渠道）
openclaw agent --message "Ship checklist" --thinking high
```

升级请看：[Updating guide](https://docs.openclaw.ai/install/updating)（并建议运行 `openclaw doctor`）。

- **beta**：预发布标签（`vYYYY.M.D-beta.N`），npm dist-tag 为 `beta`（可能不含 macOS app）。
- **dev**：`main` 分支滚动版本，npm dist-tag 为 `dev`（发布时）。

## 从源码开发

推荐使用 `pnpm` 从源码构建。`bun` 可选，用于直接运行 TypeScript。

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw

pnpm install
pnpm ui:build # 首次运行时会自动安装 UI 依赖
pnpm build

pnpm openclaw onboard --install-daemon

# 开发循环（TS 改动自动重载）
pnpm gateway:watch
```

说明：`pnpm openclaw ...` 通过 `tsx` 直接运行 TypeScript。`pnpm build` 会产出 `dist/`，供 Node 或打包后的 `openclaw` 二进制使用。

## 安全默认值（DM 私信访问）

OpenClaw 接入的是真实消息渠道。请将入站私信视为 **不可信输入**。

完整安全文档： [Security](https://docs.openclaw.ai/gateway/security)

Telegram / WhatsApp / Signal / iMessage / Microsoft Teams / Discord / Google Chat / Slack 的默认行为：

- **DM pairing**（`dmPolicy="pairing"` / `channels.discord.dmPolicy="pairing"` / `channels.slack.dmPolicy="pairing"`；旧键：`channels.discord.dm.policy`、`channels.slack.dm.policy`）：未知发件人只会收到短配对码，机器人不会处理其消息。
- 通过命令批准：`openclaw pairing approve <channel> <code>`（批准后发件人会被加入本地 allowlist）。
- 若要开放公开 DM，需显式启用：设置 `dmPolicy="open"`，并在渠道 allowlist 中包含 `"*"`（`allowFrom` / `channels.discord.allowFrom` / `channels.slack.allowFrom`；旧键：`channels.discord.dm.allowFrom`、`channels.slack.dm.allowFrom`）。

可运行 `openclaw doctor` 检查风险或错误 DM 配置。

## Tailscale 接入（Gateway dashboard）

在 Gateway 绑定 loopback 的同时，OpenClaw 可自动配置 Tailscale **Serve**（tailnet 内）或 **Funnel**（公网）。配置项为 `gateway.tailscale.mode`：

- `off`：关闭 Tailscale 自动化（默认）。
- `serve`：通过 `tailscale serve` 提供 tailnet 内 HTTPS（默认使用 Tailscale 身份头）。
- `funnel`：通过 `tailscale funnel` 提供公网 HTTPS（要求共享密码认证）。

注意：

- 启用 Serve/Funnel 时，`gateway.bind` 必须保持 `loopback`（OpenClaw 会强制）。
- 若设置 `gateway.auth.mode: "password"` 或 `gateway.auth.allowTailscale: false`，可强制 Serve 也使用密码。
- 未设置 `gateway.auth.mode: "password"` 时，Funnel 不会启动。
- 可选项：`gateway.tailscale.resetOnExit`，用于退出时撤销 Serve/Funnel。

详情： [Tailscale guide](https://docs.openclaw.ai/gateway/tailscale) · [Web surfaces](https://docs.openclaw.ai/web)

## Remote Gateway（Linux 场景）

Gateway 完全可以运行在小型 Linux 实例上。客户端（macOS app、CLI、WebChat）可通过 **Tailscale Serve/Funnel** 或 **SSH 隧道**连接；你依然可以配对设备节点（macOS/iOS/Android）执行设备本地操作。

- **Gateway 主机**默认执行 exec 工具和渠道连接。
- **设备节点**通过 `node.invoke` 执行设备本地操作（`system.run`、相机、录屏、通知）。
  简单说：exec 在 Gateway 所在机执行；设备能力在设备本机执行。

详情： [Remote access](https://docs.openclaw.ai/gateway/remote) · [Nodes](https://docs.openclaw.ai/nodes) · [Security](https://docs.openclaw.ai/gateway/security)

## 通过 Gateway 协议管理 macOS 权限

macOS app 可运行在 **node mode**，并通过 Gateway WebSocket 暴露能力与权限映射（`node.list` / `node.describe`）。客户端随后可通过 `node.invoke` 执行本地动作：

- `system.run`：执行本地命令并返回 stdout/stderr/exit code；若需屏幕录制权限请设置 `needsScreenRecording: true`（否则会得到 `PERMISSION_MISSING`）。
- `system.notify`：发送系统通知；如果通知权限被拒绝会失败。
- `canvas.*`、`camera.*`、`screen.record`、`location.get` 同样通过 `node.invoke` 路由，并受 TCC 权限状态约束。

Elevated bash（主机权限）与 macOS TCC 分离：

- 使用 `/elevated on|off` 切换每会话的提权开关（前提是启用且在 allowlist 中）。
- Gateway 会通过 `sessions.patch`（WS 方法）持久化该开关，并与 `thinkingLevel`、`verboseLevel`、`model`、`sendPolicy`、`groupActivation` 一并存储。

详情： [Nodes](https://docs.openclaw.ai/nodes) · [macOS app](https://docs.openclaw.ai/platforms/macos) · [Gateway protocol](https://docs.openclaw.ai/concepts/architecture)

## Agent to Agent（sessions_* 工具）

- 用于在不同会话之间协作，无需切换聊天渠道。

- `sessions_send`：向另一个会话发消息；可选 reply-back ping-pong + announce step（`REPLY_SKIP`、`ANNOUNCE_SKIP`）。

## 聊天命令

可在 WhatsApp / Telegram / Slack / Google Chat / Microsoft Teams / WebChat 发送（群组命令仅 owner 可用）：

- `/status`：紧凑会话状态（模型 + tokens，若可用会显示 cost）
- `/new` 或 `/reset`：重置会话
- `/compact`：压缩会话上下文（summary）
- `/think <level>`：`off|minimal|low|medium|high|xhigh`（仅 GPT-5.2 + Codex 模型）
- `/verbose on|off`
- `/usage off|tokens|full`：每条回复的 usage footer
- `/restart`：重启 gateway（群组内仅 owner）
- `/activation mention|always`：群组激活方式切换（仅群组）

## 安全模型（重要）

- **默认行为**：main 会话下工具在宿主机运行，因此当只有你自己使用时，agent 拥有完整主机访问能力。
- **群组/渠道安全**：设置 `agents.defaults.sandbox.mode: "non-main"`，让 **非 main 会话**（群组/渠道）在每会话独立 Docker sandbox 中运行；这些会话里的 bash 会在 Docker 中执行。
- **沙箱默认策略**：allowlist 包含 `bash`、`process`、`read`、`write`、`edit`、`sessions_list`、`sessions_history`、`sessions_send`、`sessions_spawn`；denylist 包含 `browser`、`canvas`、`nodes`、`cron`、`discord`、`gateway`。

详情： [Security guide](https://docs.openclaw.ai/gateway/security) · [Docker + sandboxing](https://docs.openclaw.ai/install/docker) · [Sandbox config](https://docs.openclaw.ai/gateway/configuration)

### [WhatsApp](https://docs.openclaw.ai/channels/whatsapp)

- 设备登录：`pnpm openclaw channels login`（凭据保存在 `~/.openclaw/credentials`）。
- 通过 `channels.whatsapp.allowFrom` 配置允许与助手交互的用户。
- 若设置了 `channels.whatsapp.groups`，它会变为群组 allowlist；可包含 `"*"` 放行所有群组。

### [Telegram](https://docs.openclaw.ai/channels/telegram)

- 设置 `TELEGRAM_BOT_TOKEN` 或 `channels.telegram.botToken`（环境变量优先）。
- 可选：设置 `channels.telegram.groups`（配合 `channels.telegram.groups."*".requireMention`）；设置后即群组 allowlist（可用 `"*"` 放行全部）。按需配置 `channels.telegram.allowFrom` 或 `channels.telegram.webhookUrl` + `channels.telegram.webhookSecret`。

```json5
{
  channels: {
    telegram: {
      botToken: "123456:ABCDEF",
    },
  },
}
```

### [Slack](https://docs.openclaw.ai/channels/slack)

- 设置 `SLACK_BOT_TOKEN` + `SLACK_APP_TOKEN`（或 `channels.slack.botToken` + `channels.slack.appToken`）。

### [Discord](https://docs.openclaw.ai/channels/discord)

- 设置 `DISCORD_BOT_TOKEN` 或 `channels.discord.token`（环境变量优先）。
- 可选：设置 `commands.native`、`commands.text`、`commands.useAccessGroups`，以及 `channels.discord.allowFrom`、`channels.discord.guilds`、`channels.discord.mediaMaxMb`。

```json5
{
  channels: {
    discord: {
      token: "1234abcd",
    },
  },
}
```

### [Signal](https://docs.openclaw.ai/channels/signal)

- 需要 `signal-cli` 以及 `channels.signal` 配置段。

### [BlueBubbles (iMessage)](https://docs.openclaw.ai/channels/bluebubbles)

- **推荐** iMessage 接入方式。
- 配置 `channels.bluebubbles.serverUrl` + `channels.bluebubbles.password` 以及 webhook（`channels.bluebubbles.webhookPath`）。
- BlueBubbles 服务端运行在 macOS；Gateway 可运行在 macOS 或其它主机。

### [iMessage (legacy)](https://docs.openclaw.ai/channels/imessage)

- 基于 `imsg` 的 legacy macOS-only 接入（需登录 Messages）。
- 若设置 `channels.imessage.groups`，即变为群组 allowlist；可包含 `"*"` 放行全部。

### [Microsoft Teams](https://docs.openclaw.ai/channels/msteams)

- 配置 Teams app + Bot Framework，然后添加 `msteams` 配置段。
- 通过 `msteams.allowFrom` 限制可交互用户；群组访问通过 `msteams.groupAllowFrom` 或 `msteams.groupPolicy: "open"`。

### [WebChat](https://docs.openclaw.ai/web/webchat)

- 使用 Gateway WebSocket；无需单独 WebChat 端口/配置。

浏览器控制（可选）：

```json5
{
  browser: {
    enabled: true,
    color: "#FF4500",
  },
}
```
